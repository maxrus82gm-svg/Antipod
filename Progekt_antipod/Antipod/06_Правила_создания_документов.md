# 06_Правила_создания_документов

## 1. Назначение документа

Этот документ описывает правила создания новых Markdown-документов в проекте Antipod.

Документ нужен для экономии контекста.

Агент не обязан читать этот файл перед каждой задачей.

Агент читает этот файл только тогда, когда задача создаёт новый документ или меняет структуру документации.

---

## 2. Когда читать этот документ

Агент обязан прочитать этот документ, если задача:

- создаёт новый `.md`-документ;
- создаёт новую папку документации;
- меняет структуру документации;
- предлагает новое название документа;
- переносит правила из одного документа в другой.

Если задача не создаёт новые документы и не меняет структуру документации, этот файл читать не нужно.

---

## 3. Главное правило

Перед созданием нового документа агент обязан явно указать, какие документы нужно прочитать для этой конкретной задачи.

Пример:

```text
Для создания нового документа нужно прочитать:
- Старт.md
- Documentation/Агентский_цикл.md
- 06_Правила_создания_документов.md
```

После этого агент создаёт документ только в рамках задачи.

---

## 4. Русские названия документов

Все основные документы проекта называются на русском языке.

Англоязычные названия использовать только если это явно нужно по технической причине.

---

## 5. Формат документа

Новый документ должен быть обычным Markdown-файлом.

Требования:

- расширение `.md`;
- понятный русский заголовок;
- короткие разделы;
- удобное чтение в Obsidian;
- без лишнего HTML;
- без автогенерированного мусора;
- без повреждения кириллицы.

---

## 6. Кодировка

Все новые Markdown-документы сохранять в:

```text
UTF-8 без BOM
```

Если в документе есть кириллица, агент обязан убедиться, что русский текст читается нормально и не повреждён.

Запрещено заменять кириллицу на:

- вопросительные знаки;
- кракозябры;
- транслит;
- нечитаемые символы.

---

## 7. Связь с другими документами

Если новый документ становится важной частью карты проекта, агент добавляет ссылку на него в подходящее место.

Для текущей структуры основной центр графа:

```text
00_Главная.md
```

Не добавлять ссылку во все документы сразу без необходимости.

---

## 8. Экономия контекста

Этот документ не является обязательным для чтения перед каждой задачей.

Его нужно читать только при создании новых документов или изменении структуры документации.

Главное правило:

```text
Читать только то, что нужно для текущей задачи.
```

---

## 9. Отчёт после создания документа

После создания нового документа агент указывает в отчёте:

```text
Созданный документ:
- путь

Зачем создан:
- причина

Какие документы нужно читать в будущем:
- список

Кодировка:
- UTF-8 без BOM
- кириллица проверена
```
