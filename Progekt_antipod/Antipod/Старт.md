# Старт

Это стартовый документ проекта Antipod для агентской работы.

Перед началом любой задачи агент читает:

```text
Старт.md
Documentation/Задача_агенту.md
```

Если задача относится к отдельной теме, агент дополнительно читает только нужные тематические правила из канонических папок документации.

Канонические пути документации:

```text
Documentation/
Documentation/Rules/
Documentation/Project_Docs/
Documentation/Tasks/
Documentation/История_задач.md
```

Если нужного документа или правила нет, агент указывает это в отчете и не делает вид, что документ существует.

## Правило создания новых документов

Если задача создаёт новый Markdown-документ или меняет структуру документации, агент обязан прочитать:

```text
06_Правила_создания_документов.md
```

При обычных задачах без создания новых документов этот файл читать не нужно.

## Правило Git / Review

Если задача связана с Git, GitHub, commit, push, pull, diff, branch, историей изменений или review, агент обязан прочитать:

```text
07_Правила_Git_и_Review.md
```

При обычных задачах без Git этот документ читать не нужно.

Git полностью контролирует пользователь.

Агент не выполняет `commit`, `push`, `pull`, `merge`, `rebase`, `reset`, `clean`, `checkout`, `switch`, создание веток или любые другие Git-действия, которые меняют состояние репозитория.

## Правило GitHub-review после отчёта агента

Если пользователь сообщает, что агент выполнил задачу, или просит REVIEW результата, ассистент не должен опираться только на текстовый отчёт агента.

Ассистент обязан сам проверить GitHub-репозиторий по правилам `07_Правила_Git_и_Review.md`.

Ассистент обязан явно вывести статус чтения репозитория:

```text
Repository:
Branch/ref:
Commit SHA:
Commit message:
Files read:
Проверка:
```

Если изменения не найдены на GitHub, ассистент пишет:

```text
Push не обнаружен.
Локальные изменения не проверялись.
```

Git полностью контролирует пользователь.

Ассистент не выполняет commit, push, pull, merge, rebase, reset, clean, checkout, switch, создание веток или изменение истории Git.

Это правило применяется только к REVIEW, Git, GitHub, commit, push, отчётам агента, проверке выполненных задач и истории изменений.

Для обычных обсуждений, исследований и планирования GitHub читать не нужно.

Основной принцип работы:

```text
маленькая задача
понятное изменение
проверка
отчет
история
следующий шаг
```
