# ПРАВИЛО БОЛЬШОГО И МАЛОГО БЛОКА

## Большой блок

Большой блок:

- хранится в `Documentation/Задача_агенту.md`;
- остаётся основной задачей в документе;
- содержит цель, контекст, scope, разрешённые файлы, запреты и проверки;
- не переписывается заново только из-за поздних уточнений.

## Малый блок

Малый блок:

- отправляется агенту в чат;
- содержит команду прочитать актуальную задачу из `Documentation/Задача_агенту.md`;
- может содержать список дополнительных документов для чтения;
- может содержать уточнения, появившиеся после подготовки большого блока;
- может содержать критичные напоминания;
- дополняет большой блок, но не заменяет его.

Если уточнение появилось после подготовки большого блока и до отчёта агента, ChatGPT добавляет это уточнение в малый блок автоматически, без переписывания большого блока.

Если малый блок противоречит большому блоку, агент обязан остановиться и указать конфликт в отчёте.

# КОМПАКТНЫЙ ОТЧЁТ АГЕНТА

Последний отчёт агента хранится в `Documentation/Задача_агенту.md` в блоке `БЛОК 2 — ПОСЛЕДНИЙ ОТЧЁТ АГЕНТА`, ниже активной задачи.

Перед записью отчёта агент сверяет фактическое состояние файлов через безопасные Git-проверки чтения, если Git использовался в задаче.

Если `git status`, `git diff --name-only`, `git ls-files --others --exclude-standard` или другая безопасная Git-проверка чтения показывает файл как изменённый, добавленный, удалённый или untracked, этот файл обязательно указывается в отчёте.

Файлы разделяются так:

- `CHANGED` — намеренно изменённые файлы в рамках задачи;
- `UNEXPECTED CHANGES` — побочные, автоматические или не входившие в задачу изменения;
- `UNTRACKED` — новые файлы, которые ещё не отслеживаются Git.

Побочные изменения нельзя скрывать. Если, например, `.obsidian/workspace.json` изменился автоматически из-за Obsidian, но задача не разрешала менять его намеренно, файл указывается в `UNEXPECTED CHANGES`. Если файл уже отслеживается Git и это создаёт риск, риск указывается в `PROBLEMS`.

Формат отчёта:

```text
DONE:
- что сделано

CHANGED:
- намеренно изменённые файлы

UNEXPECTED CHANGES:
- побочные или автоматические изменения

UNTRACKED:
- новые неотслеживаемые файлы

CHECKS:
- что проверено

GIT:
- commit не создавался агентом
- push не выполнялся агентом
- pull не выполнялся агентом
- ветки не создавались и не переключались агентом
- Git-история не изменялась агентом

PROBLEMS:
- реальные проблемы или риски

NEXT:
- следующий шаг
```

## Правило очистки активной задачи после отчёта

После успешного выполнения задачи агент обязан очистить `БЛОК 1 — ТЕКУЩАЯ ЗАДАЧА`.

После очистки `БЛОК 1` должен содержать только:

БЛОК 1 — ТЕКУЩАЯ ЗАДАЧА

Активной задачи нет.  
Предыдущая задача выполнена.  
Ожидается новая задача от пользователя.

Если задача не выполнена, есть критичная ошибка или требуется решение пользователя, `БЛОК 1` не очищать. В этом случае агент обязан явно указать причину в `PROBLEMS`.

Очистка `БЛОКА 1` выполняется только после того, как отчёт записан в `БЛОК 2 — ПОСЛЕДНИЙ ОТЧЁТ АГЕНТА`.
