# БЛОК 1 — ТЕКУЩАЯ ЗАДАЧА

TASK A-005 — Закрепить правило большого и малого блока для задач агента

## Цель

Закрепить порядок постановки задач агенту:

- большой блок хранится в документе задачи;
- малый блок отправляется агенту в чат;
- малый блок может содержать уточнения, появившиеся после подготовки большого блока;
- если уточнение появилось до отчёта агента, ChatGPT добавляет его в малый блок автоматически, без переписывания большого блока.

Также нужно:

- переименовать документ `Агентский_цикл.md` в `Задача_агенту.md`, если это безопасно;
- укоротить формат отчёта агента;
- закрепить место хранения последнего отчёта агента.

## Уточнение из малого блока

Большой блок должен оставаться основной задачей в документе, а малый блок в чате может дополнять его новыми уточнениями.

Если уточнение появилось после подготовки большого блока, ChatGPT добавляет его в малый блок, чтобы не переделывать большой блок заново.

Предпочтительное новое название документа:

```text
Задача_агенту.md
```

Git контролирует только пользователь. Commit, push, pull, ветки, откаты и изменение истории Git не выполнять.

# ПРАВИЛО БОЛЬШОГО И МАЛОГО БЛОКА

## Большой блок

Большой блок:

- хранится в `Documentation/Задача_агенту.md`;
- остаётся основной задачей в документе;
- содержит цель, контекст, scope, разрешённые файлы, запреты и проверки;
- не переписывается заново только из-за поздних уточнений.

## Малый блок

Малый блок:

- отправляется агенту в чат;
- содержит команду прочитать актуальную задачу из `Documentation/Задача_агенту.md`;
- может содержать список дополнительных документов для чтения;
- может содержать уточнения, появившиеся после подготовки большого блока;
- может содержать критичные напоминания;
- дополняет большой блок, но не заменяет его.

Если уточнение появилось после подготовки большого блока и до отчёта агента, ChatGPT добавляет это уточнение в малый блок автоматически, без переписывания большого блока.

Если малый блок противоречит большому блоку, агент обязан остановиться и указать конфликт в отчёте.
